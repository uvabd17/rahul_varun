01 — Premise

What this app is, in one paragraph.

Five friends — Alice, Bob, Charlie, David, and Emma — go out and buy things together. One person pays the bill. Everyone who was in on it owes their share back. The app remembers every expense, then works out who owes whom.

It's a small web page you open by double-clicking index.html. There's no server, no login, no database. Everything lives inside your browser.

Web page A document made of HTML (structure), CSS (looks), and JavaScript (behaviour). Your browser reads all three and paints them onto the screen.
Browser The program you use to visit websites — Chrome, Firefox, Safari. It runs HTML, CSS, and JavaScript.
02 — Mental model

The one big idea.

Every page in the app is a different way of looking at the same list. That list is your expenses, and it lives in a corner of the browser called localStorage.

localStorage A small storage box built into every browser. You can put text into it under a name, and it stays even if you close the tab or shut the computer down. Each website has its own box.

Save, edit, or delete an expense — it changes the list. Then every page you visit shows you the fresh answer, because they all read from the same list.

  2. Amount is a number greater than zero.
  3. Someone was picked as the payer.
  4. A category was chosen.
  5. A date was picked.
  6. At least one member is ticked and the payer is among them.

The form calls a helper called addExpense. That helper does three quiet things: it reads the current list, appends the new expense with a fresh id, and writes the whole list back out.

const addExpense = (obj) => {

  return id;

Spread (...obj) "Unpack every field of obj right here." So { id, ...obj } means "a new object with id plus everything obj already had."
6

Every open page hears the news.

After writing to storage, saveExpenses shouts an announcement into the page:

const saveExpenses = (list) => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(list));
  window.dispatchEvent(new CustomEvent('expenses-updated'));
};
js/storage.js

Every page that shows data has already asked to be told when this happens:

06 — The math, part one

How the split works.

A split is one line of arithmetic: amount ÷ number of people who shared it = share per person.

Alice paid the full ₹900. Her own share is ₹300, so she is effectively out of pocket by ₹600. The other ₹600 is what she's owed:

  • Bob owes Alice ₹300.
  • Charlie owes Alice ₹300.

In code, the same idea is one line inside a loop that runs over every expense:

const share = Number(e.amount) / e.sharedBy.length;
e.sharedBy.forEach(m => {
  if (m !== e.paidBy) owes[m][e.paidBy] += share;
});

07 — The math, part two

How the balances work.

Once several expenses exist, the app looks at each member and asks two questions:

  • Paid — how much money did they hand over in total?
  • Share — how much did they actually consume, summed across every expense they were in?

Then it subtracts one from the other:

Balance = Paid − Share

A positive balance means the group owes you. A negative balance means you owe the group. Loading all six sample expenses gives:

MemberPaidShareBalance
Alice ₹900 ₹600 +₹300
Bob ₹1,800₹1,150+₹650
Charlie ₹500 ₹1,150−₹650
David ₹400 ₹650 −₹250
Emma ₹300 ₹350 −₹50

The code that produces this table is small and readable:

const computeBalances = (expenses) => {
  const paid  = Object.fromEntries(MEMBERS.map(m => [m, 0]));
  const share = Object.fromEntries(MEMBERS.map(m => [m, 0]));

  expenses.forEach(e => {
    paid[e.paidBy] += Number(e.amount);
    const s = Number(e.amount) / e.sharedBy.length;
    e.sharedBy.forEach(m => { share[m] += s; });
  });

  return MEMBERS.map(m => ({
    member:  m,
    paid:    paid[m],
    share:   share[m],
    balance: paid[m] - share[m]
  }));
};
js/balance.js · computeBalances
Object.fromEntries + map A two-step trick: MEMBERS.map(m => [m, 0]) turns the names list into pairs like ['Alice', 0]. Object.fromEntries then folds those pairs into { Alice: 0, Bob: 0, … } — a running total for every member, starting at zero.
+= Short for "add this to what's already there." paid[e.paidBy] += 900 means "increase Alice's paid total by 900."
08 — The math, part three

Who owes whom, without the noise.

A naïve list of debts would tell you that Alice owes Bob ₹300 for the Biryani and Bob owes Alice ₹300 for the Pizza. Two payments, but they cancel out. The app nets those out before showing you anything.

Between every pair, it looks at what A owes B and what B owes A, subtracts one from the other, and keeps only the leftover in the direction it belongs.

for (let i = 0; i < MEMBERS.length; i++) {
  for (let j = i + 1; j < MEMBERS.length; j++) {
    const a = MEMBERS[i], b = MEMBERS[j];
    const diff = owes[a][b] - owes[b][a];
    if      (diff >  0.005) debts.push({ from: a, to: b, amount: diff });
    else if (diff < -0.005) debts.push({ from: b, to: a, amount: -diff });
  }
}

10 — Vocabulary

Every term in one place.

HTML
HyperText Markup Language. The tags (<div>, <p>, <input>) that describe what's on the page.
CSS
Cascading Style Sheets. The rules that decide what things look like — colour, size, spacing, layout.
JavaScript
The programming language that runs in the browser. It handles clicks, updates the page, does the maths.
ES6
A modern version of JavaScript (also called ES2015). It added the arrow function (=>), const, template literals, and other pieces the code relies on.
DOM
Document Object Model. The live tree of every element on the page. Changing the DOM changes what you see.
localStorage
A small storage box per website, built into every browser. Text goes in under a key; text comes out later — even after the tab closes.
JSON
JavaScript Object Notation. A text format for objects and lists. JSON.stringify(list) turns data into text; JSON.parse(text) turns it back.
Event / Event listener
An event is something that happens (a click, a submit, a change). A listener is a function set up to run when a particular event happens.
Custom event
An event we invent ourselves ('expenses-updated') and broadcast so any interested code can react.
Array
An ordered list, written with square brackets: [1, 2, 3].
Object
A bundle of labelled values, written with curly braces: { name: 'Pizza', amount: 900 }.
const
Declares a value that never changes.
Arrow function
A shorthand for writing a function: (x) => x * 2.
Template literal
A string in backticks with ${…} placeholders that are filled in with values.
Spread (...)
Unpacks a list or object into another. [...list, newItem] makes a new list containing everything plus the new item.
map / filter / reduce / forEach
The four array methods the app leans on. map transforms each item into a new one. filter keeps only items that match. reduce boils a list down to one value (a total, for instance). forEach just runs code on each item without returning anything.
Chart.js
A small charting library loaded from a CDN on the Reports page. It draws the pie chart and the bar chart.
CDN
Content Delivery Network. A public server that hosts libraries (like Chart.js) so pages can load them without shipping their own copy.
Fin

That's the whole app.

One list in localStorage. Five pages that read it. Two small functions in balance.js that turn the list into an answer. Everything else is arrangement.
